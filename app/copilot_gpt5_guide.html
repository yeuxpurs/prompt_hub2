2. 실전 레시피 12선 + 예시 프롬프트

💡 아래 프롬프트는 그대로 복사·붙여넣기 후 대괄호만 교체하면 됩니다

🎤 A. 회의·음성

1. 실시간 동시통역 & 노트

역할: 회의 동시통역과 기록 보조.
언어: 한국어↔영어 양방향, 용어집은 [첨부/아래 블록].
지시: 핵심 결정·주요 수치·담당자·데드라인을 실시간으로 캡처하고,
     15분 간격 요약과 회의 종료 시 액션 아이템(담당/기한) 목록을 보여줘.
형식: [표 + 불릿]으로 출력, 모호한 내용은 즉시 확인 질문.

2. 고객 콜 코칭(음성)

내 발화를 간결/공감/해결지향 톤으로 리라이팅해서
마이크로세컨드 지연 없이 제안해줘.
"금지 표현" 목록은 [붙임] 준수. 고객 감정 레벨 3단계로 태깅.

🖼️ B. 이미지 이해/변환

3. 스크린샷 → 버그 티켓

[스크린샷 N장]을 분석해 재현 단계, 예상/실제 동작, 환경정보를 추출하고 
JIRA 템플릿에 맞춰 요약해줘. 심각도·우선순위 제안과 원인 가설 3개 포함.
산출물: [Markdown + 코드블록 재현 스크립트(가능 시)].

4. 도표·사진 → 데이터 테이블 + 인사이트

첨부 이미지들에서 표/수치/OCR을 표준화된 CSV로 추출하고,
지표 5개(KPI)와 시사점 5개를 도출해줘.
형식: (1) CSV, (2) 요약(불릿), (3) 다음 실험 제안 3가지.

🎨 C. 창작(이미지)

5. 브랜드 이미지 캠페인 세트

목표: [제품/서비스] 런칭용 이미지 4종 세트 생성.
세트: (1) 1:1 히어로, (2) 16:9 웹 배너, (3) 4:5 SNS 피드, (4) 9:16 스토리.
스타일: [브랜드 톤/팔레트/참고 링크 요약].
피사체/장면: [핵심 메시지]가 바로 보이게 간결하게.
출력: 각 버전에 대한 프롬프트/시드설명 + 접근성용 alt text 1문장.
제약: 상표/인물권 제한 준수, 과장·허위 효능 금지.

6. 제품 렌더 합성(컨셉)

입력: 제품 정면/측면 이미지 2장.
지시: 배경 제거 후 [상황/로케이션]에 사실적으로 합성, 그림자/반사 정합.
출력: 전/후 비교, 색상 3안, 인네일 256px/웹 1600px 패키지 설명.

📊 D. PowerPoint

자료: [PDF/문서/링크 목록].
과제: 12슬라이드 투자자 피치.
구성: 1표지→2문제→3기회→4솔루션→5데모→6시장→
     7경쟁→8비즈니스모델→9실적→10로드맵→11팀→12요청
지시: 각 슬라이드 3불릿 이하, 발표자노트 80-120자
톤: 간결·데이터 중심

📄 E. Word/보고서

입력: RFP 원문(필수 요구/평가기준 강조).
목표: 8쪽 제안서 초안.
섹션: 요약/요구해석/솔루션/계획/인력/리스크/가격
지시: 평가기준 가중치 맞춘 가독성, 
     "필수 요건 충족 매핑" 표 포함

📈 F. Excel/데이터

시나리오: [기준 가정 테이블]로 CAC/LTV/BE 산출.
지시: 민감도(±10/20/30%) 테이블과 토네이도 차트,
     임계값/의사결정 규칙 5개,
     마지막에 요약 5문장

📧 G. 이메일/고객 커뮤니케이션

현재 스레드를 5불릿으로 요약.
고객 페인포인트/요구/제약 추출.
답장 옵션 3개: (1)정중/신속, (2)대안제시, (3)강한마감유도.
각 옵션에 '다음 단계' 3줄 포함

3. "멀티모달 + 문서화" 풀파이프라인 예시

📄 이미지 촬영 → 분석 → 문서/슬라이드 생성 → 후속 메일 한 번에

1단계: 현장 사진 5장 업로드
"각 사진에서 결함 유형/위치/심각도(1—5)를 표로 정리, 근거를 확대 캡처와 함께 첨부. 중복 결함은 병합."
2단계: 자동 PPT 생성
"위 표를 기반으로 품질 리뷰 8슬라이드 작성. 경향 그래프 1개, 재발 방지 대책 3개, 담당/기한 매핑표 포함."
3단계: 요약 메일 초안
"슬라이드 요점을 6줄로 요약해 고객에게 보내는 메일로 작성. 톤: 사실·책임감·구체적 일정."

4. 프롬프트 설계 체크리스트 (팀 공통 규칙)

🎯 목표·제약

"이번 산출물의 성공 기준은 무엇인가?"를 첫 줄에 명시

🔍 입력 범위

사용할 파일/대화/회의를 정확히 지정 ("현재 문서만", "첨부 3개만")

📋 형식 고정

표/불릿/코드/슬라이드 수/노트 길이를 미리 합의

📚 근거 요구

"숫자·주장에는 근거/출처/불확실성"을 붙이도록 지시

✅ 검증 단계

"마지막 슬라이드(혹은 장)에 검증 체크리스트/리스크" 삽입

🔄 재사용 변수화

날짜·브랜드 팔레트·톤·금지 표현을 팀 템플릿으로 관리

5. 가드레일 & 운영 팁

⚠️ 개인정보/상표/초상권

이미지 생성 시 실존 인물/로고/상표 회피 지시. 합성 시 권리 확인 문구 포함.

🔍 사실 검증

외부 통계·수치가 들어가면 "출처 각주"를 요구하고, 중요한 결론은 대안/반례와 함께 제시.

♿ 접근성

자동으로 alt text/캡션을 요청(슬라이드·이미지 공통).

🔄 리비전 루프

"슬라이드 n만 재작성", "톤만 바꾸기", "근거 강화"처럼 부분 리젝을 습관화.

6. 미니 프롬프트 라이브러리

이 섹션은 향후 팀의 공통 프롬프트를 추가하기 위해 비워져 있습니다.
